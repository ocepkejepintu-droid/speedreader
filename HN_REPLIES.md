# HN Reply Templates (3 bullseye targets)

These are templates. Customize to the actual thread OP before posting. Each is under 120 words. No emoji, no "great post," no "as a X I think" openers. Each ends with a soft offer to share more — no link unless the thread is a Show HN / Ask HN asking for tools.

## Reply A — "What apps do you use daily?" (Show & tell / setup thread)

Customize to the actual thread before posting.

> I keep my daily setup almost embarrassingly small: a text editor, a terminal, a browser tab open on my RSS reader, and a PWA EPUB reader for the 30 minutes on the train. The combination works because none of them try to be a feed.
>
> The thing I learned the hard way: an "app I use daily" isn't the one with the most features, it's the one with the lowest activation cost. A 200ms tap to start reading is a 50% chance I'll actually read. A 5-second "open Kindle, sync, find the book, find the line" is a 0% chance.
>
> Happy to share the full list if anyone's curious.

## Reply B — "Why do you read so slowly?" (reading habits thread)

Customize to the actual thread before posting.

> "Slow" is doing a lot of work in that question. The honest answer for me was that I wasn't reading slowly, I was reading in a format that lets me skim. Skimming feels like reading, retention included.
>
> Switching to an RSVP reader (one word at a time, anchored on the same focal point) was the unlock. Not because it makes me faster — for dense nonfiction, comprehension peaks around 350-450 WPM no matter what — but because it kills the skim reflex. There's nothing to skim to. You finish the chapter you started.
>
> If anyone wants the comprehension-vs-WPM data I leaned on, happy to dig it up.

## Reply C — "Show HN: a new text editor" or "Ask HN: tools for focus" (minimal-tools thread)

Customize to the actual thread before posting.

> The constraint I landed on after building a few of these: minimal stops being helpful the moment you add a settings panel. The first version of my own tool had three settings. The fifth version had forty. The fifth was worse.
>
> A trick that worked: settings that adjust themselves based on the content. A 200-word article gets a different WPM default than a 90,000-word novel, automatically. No "pick your WPM" onboarding step. The tool reads the room.
>
> I built a small PWA reader in this same spirit. If anyone wants to poke at the architecture, I'm around.
